Webcite

RefMe, a citation website, was shutdown and merged into CiteThisForMe The merged website was not as responsive and did not support saving your bibliographies. Of course, these features were included in the premium version but I was reluctant to pay for a citation service. As a result, I decided to create my own version that would scrape information from websites and create a bibliography that can be saved online.

The core web scraping was done by requesting the source file of the URL and looking for metadata such as the author and date modified of a website. It is fairly straightforward except for the inclusion of a logistic regression learning algorithm used to 'find' the author when the author is not explicitly stated. The inclusion of this logistic regression learner was just for fun. The learner was trained with 7 features of text in links found on the website: inclusion of numbers; amount of 'words' with only alphabets; whether there is a first name matching the contents of a csv file; whether there is a surname matching the contents of a csv file; whether there is a word matching the contents of a csv file of common words used; length of the text; how many words are capitalised. The trained parameters were fairly accurate, although it can get confused when there are other names that are not related to the author of a website.

Here is the web scraping in action on an scripts version of webcite:

After finishing the core web scraping program, I moved on to creating a wrapper using HTML and making a webapp. I tried implementing this in many different Python frameworks such as Django and Flask but my progress was slow as this was my first time creating a webapp. Being impatient, I decided to take the easiest route and use PHP. This made it slightly more complicated to use the core web scraper which was written in Python and modifications had to made. A Python file acts as the interface between the webapp and the website written in PHP. Different parameters can be passed onto this file for it to do different things, allowing the PHP to run the Python file using exec and passing necessary arguments. Another choice I made was to not use any existing PHP framework such as Laravel as I would learn a lot more. This also let me know how everything works instead of blindly following a tutorial listing out functions from a library. However, this made my code extremely messy and would make implementing CSS significantly harder.

When trying to implement CSS, I realised that using an existing framework such as Boostrap would be almost impossible as I had to rewrite a significant portion of my code. Consequently, I decided to simply write my own stylesheets. Still, I had to rewrite part of my code which included adding divs and tables in order to structure the content of the webpage. I did not make use of flexbox as I was still unfamiliar with flexbox and wished to create something visually appealing fast.

Calc

I created this small project as an attempt to familiarise myself with the use, traversal and manipulation of tree structures. Mathematical expressions are can be stored recursively in trees, which makes them suitable for this project. Each mathematical expression is a tree structure that (usually) has an operation as a root node. When evaluating the result of the expression, the program uses the value of the child nodes and the nature of the operation at the root node.

Example: 1+(2*(3+4)+5)*6-7

alienonolympus.github.io

This website was made to record the projects I have done and also keep track of my current extracirricular projects. A lot of the time I would start a project and forget about it, thus recording all my ongoing projects will allow me to remember to work on it. I would also use this portfolio as a 'diary' of my programming progress, allowing me to look back at decisions I made during the development process and evaluate them if there are any major issues popping up.

I originally used markdown and chose a template from Github Pages. However, I decided to change to using HTML and a premade CSS template due to the lack of flexibility of markdown. Pure CSS was chosen becaues its website was the one that first one to finish loading when I opened a myriad of CSS templates. Its minimalist design suited this portfolio due to this portfolio's simplicity.

Afterwards, I switched to using a Dark Theme made using Materialize CSS. I felt like the new theme represented the nature of my portfolio better while being more aesthetically pleasing. In addition, everything is located one one page, saving time when loading a separate page. However, the new theme is not optimised for mobile at all, which is something I need to work on at a future date.
